Использование скриптов Substrate

Substrate-up скрипты позволяют настроить готовые к работе узлы substrate или модули исполняющей среды.

Чтобы использовать эти скрипты, убедитесь, что Вы завершили Ускоренную установку для запуска Substrate.

Эти скрипты обновляются время от времени, поэтому перед запуском их локально убедитесь, что они обновлены, запустив:

f=`mktemp -d`
git clone https://github.com/paritytech/substrate-up $f
cp -a $f/substrate-* ~/.cargo/bin
cp -a $f/polkadot-* ~/.cargo/bin

substrate-node-new

Скрипт substrate-node-new загружает и компилирует копию шаблона узла Substrate. Это даст Вам готовый к работе узел Substrate с шаблонным модулем исполнения.

Выполните команду substrate-node-new с следующими параметрами:

substrate-node-new <node-name> <author>

Где:

  • <node-name> является именем для Вашей среды исполнения Substrate. Это обязательный параметр.

  • <author> показывает людей или команду, которые поддерживают эту среду исполнения узла. Это обязательный параметр.

После запуска команды substrate-node-new, для завершения компиляции потребуется несколько минут (в зависимости от Вашего оборудования).

substrate-module-new

После того, как Вы запустите локальным узел, используя скрипт substrate-node-new, вы можете добавить дополнительные модули в Вашу среду исполнения, используя скрипт substrate-module-new.

Скрипт substrate-module-new создает новый модуль исполнения на основе этого шаблона. Это даст Вам готовый к работе модуль исполнения с всеми необходимыми импортами, точками ввода и примерами тестов. Мы рекомендуем использовать этот скрипт для создания новых модулей, особенно для пользователей, которые только начинают знакомиться с Substrate, так как он также даёт Вам хорошую информацию о том, как структурирован типичный модуль исполнения Substrate.

Для использования скрипта substrate-module-new,

  1. Убедитесь, что Вы установили Substrate, имеете локальную среду исполнения, созданную с помощью substrate-node-new.

  2. Перейдите в каталог среды исполнения узла, запустив cd runtime/src.

  3. Выполните команду substrate-module-new с следующими параметрами,

    substrate-module-new <module-name>

Где:

  • <module-name> это имя для нового модуля. Это обязательный параметр.

Это создаст новый файл с именем <module-name>.rs в рабочем каталоге. Для использования этого модуля необходимо добавить ссылки на этот модуль в файл lib.rs Вашей среды исполнения узла.

Например, если Вы создаете модуль как,

substrate-module-new mymodule

Добавьте следующую строку в lib.rs, чтобы этот модуль инициализировался в Вашей среде исполнения,

mod mymodule;

Кроме того, внедрите определения модуля для Вашего модуля в lib.rs

impl mymodule::Trait for Runtime {
    // добавить необходимые типы здесь
    type Event = Event;
}

И наконец, добавьте этот модуль в construct_runtime макрос в lib.rs,

construct_runtime!(
    pub enum Runtime with Log(InternalLog: DigestItem<Hash, Ed25519AuthorityId>) where
        Block = Block,
        NodeBlock = opaque::Block,
        UncheckedExtrinsic = UncheckedExtrinsic
    {
        System: system::{default, Log(ChangesTrieRoot)},
        Timestamp: timestamp::{Module, Call, Storage, Config<T>, Inherent},
        Consensus: consensus::{Module, Call, Storage, Config<T>, Log(AuthoritiesChange), Inherent},
        Aura: aura::{Module},
        Indices: indices,
        Balances: balances,
        Sudo: sudo,
        MyModule: mymodule::{Module, Call, Storage, Event<T>},
    }
);
