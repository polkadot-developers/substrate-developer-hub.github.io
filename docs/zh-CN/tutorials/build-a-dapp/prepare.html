准备构建一个 dApp

编译 Node Template

在完成创建您的第一条 Substrate 链教程后，您的计算机上应该已编译了v2.0.0-rc4版本的Substrate Node Template。 如果您尚未完成，请完成该教程。

有经验的开发者如果想跳过这个教程来安装 Node Template，可以参考 readme 文档里的说明步骤。

安装 Front-End Template

创建您的第一个 Substrate 链教程使用了 front-end template， 因此，如果您已经完成了该教程，就不需要 额外的设置。

如有需要，请直接参考《创建您的第一条区块链》教程的 前端设置说明。

关于存在证明

我们将构建的 dapp 是一个存在证明服务（PoE）。 来自 维基百科:

存在证明是一个在线服务，通过比特币区块链中的时间戳交易验证一个计算机文件是否在特定时间之前存在。

不需要上传整个文件到区块链来“证明其存在”， 用户可以提交一个 文件哈希，称为文件摘要或校验和。 这些哈希是非常有用的，因为巨型文件可以用一个小的哈希值来唯一表示，这个值对于存储在区块链上是非常有效率的。 任何拥有原始文件的用户可以简单地通过重新计算文件的哈希值并与区块链上存储的值相比较来证明该文件与区块链上的是一致的。

为此，区块链系统还提供了一个强大的身份系统。 所以当文件摘要存储进区块链时，我们也同时记录是哪个用户上传了该摘要。 这使得该用户在后续能够证明他是该文件的拥有人。

接口与设计

我们的 PoE API 将暴露两个可调用的功能：

  • create_claim - 允许用户通过上传文件摘要来声称文件存在。
  • revoke_claim - 允许当前的所有者撤销他们的所有权。

为了实现这一点，我们只需要储存已经声称的证据以及谁提出了这些的证据即可。
