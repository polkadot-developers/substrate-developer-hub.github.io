介绍

Substrate 区块链开发框架中的一个基本特性就是支持 runtime 无分叉升级。 无分叉升级是通过区块链自身安全支持的功能，来增强区块链 runtime 的一种手段。 区块链的 runtime 定义了区块链的 状态 ，以及影响状态变化的逻辑。 Substrate支持在不需要 硬分叉 的情况下，部署新的 runtime (包括 与上个版本不兼容的 更新) 功能。 由于 runtime 定义本身就是 Substrate 链状态的一部分，所以网络参与者可以通过 extrinsic 的 set_code 函数 来更新 runtime 的状态值。 由于节点更新到 runtime 状态受区块链的共识机制和密码学相关的一些限制，网络参与者可以使用区块链就能实现无需私下信任就能更新或扩展 runtime 逻辑，而不需要分叉该链或发布一个新的区块链客户端。

本教程将使用 Substrate 开发者中心的 Node Template 来演示两个基于 FRAME的 runtime 无分叉升级机制 。 首先，使用 Sudo 模块里的 sudo_unchecked_weight 函数来添加Scheduler 模块。 然后，使用 Scheduler 模塊中的 schedule 函数來增加网络帐户的 最小余额 的升级。

如果您在本教程中遇到问题，Substrate 社区中有很多帮助资料可供参考。 我们维护了一个活跃的 #SubstrateTechnical 聊天室 并关注 Stack Overflow上的 Substrate 标签。 还可以在 subport Github仓库 上提交問題。
