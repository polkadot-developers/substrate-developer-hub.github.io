总览

欢迎来到Substrate区块链开发的美妙世界！ 这里是Substrate 知识库，专门为Substrate 开发者开放的官方文档中心。 此资源的目的是能够帮助读者了解关于Substrate区块链开发的跨学科领域知识。 This guide is broken down into several sections that explain the principles and design decisions that Substrate is built on as well as the specific skills needed to be an effective Substrate blockchain developer.

所需的一些专业知识

为了能最大限度从 Substrate中获取想要的东西，你应该预先要对计算机科学及基础区块链概念有较为扎实的基础， 对区块头、区块、客户端、哈希、交易和签名等之类的术语比较熟悉。 Substrate 是建立在 Rust 这门编程语言基础上的， 它利用新颖的设计模式来实现安全、快速的代码开发。 虽然并不需要预先了解 Rust 才能开始学习 Substrate，但是如果对 Rust 有一个很好的理解，会帮助你成为一个更优秀的 Substrate 开发者。 可查看Rust社区提供的优质资源，以构建你的Rust开发技能。

Substrate 采用了模块化的区块链开发模式，并定义了一组丰富的基础类型，使开发者不仅能有效利用它的强大开发能力，又感觉它和自己熟悉的编程语言差别不大。

架构

Substrate 客户端是一个用于运行 Substrate 区块链节点的应用程序 - 它由几个组件组成，包括但不限于：

  • 存储 是用来维持区块链这个去中心化系统所呈现的状态演变的一贯性。 区块链网络可以让所有参与者在无需信任的条件下，而对区块链存储状态达成共识。 Substrate 提供了一种简单而高效的 键值对存储机制.
  • Runtime 逻辑定义了区块的处理方式，其中包括了状态转换的逻辑。 In Substrate, runtime code is compiled to Wasm and becomes part of the blockchain's storage state - this enables one of the defining features of a Substrate-based blockchain: forkless runtime upgrades. 另外，Substrate客户端也可以拥有一个非wasm格式的"native runtime" ，它的编译语言应与客户端的编写语言一致。 客户端中向 runtime 发送可调用函数的组件被称为执行器 ，是由它来负责在native代码和编译后的Wasm之间进行选择。 Although the native runtime may offer a performance advantage, the executor will select to interpret the Wasm runtime if it implements a newer version.
  • 点对点网络 允许客户端与其他网络参与者进行通信。 Substrate 使用 libp2p 网络栈。
  • 共识 引擎提供了一种逻辑，能使网络参与者就区块链的状态达成一致。 Substrate 支持提供自定义的共识引擎，同时也提供了几个建立在 Web3 基金会研究之上的共识机制。
  • RPC(远程过程调用) 功能使用户能与区块链网络进行交互。 Substrate 提供了 HTTP 和 WebSocket RPC 两种服务器。
  • 遥测 度量通过嵌入式 Prometheus 服务器的方式对外展示。

使用方法

可以通过以下三种方式来使用Substrate：

  1. 通过 Substrate 节点： 你可以运行预先设计好的 Substrate Node 并配置它的创世区块。 在这种情况下，你只需要提供一个 JSON 文件来启动你自己的区块链。 该JSON 文件允许你配置 Substrate 节点的runtime 组成模块的初始状态，如：Balances，Staking，和 Sudo等模块。 你可以在创建你的第一条 Substrate 区块链和启动一个私有网络教程中了解更多关于运行 Substrate 节点的信息。

  2. With Substrate FRAME: You can easily create your own custom runtime using FRAME (Framework for Runtime Aggregation of Modularized Entities), which is what is used to build the Substrate Node. 这就为你的区块链逻辑开发提供了巨大的自由度，允许你配置数据类型，从模块库(称为 "pallets") 中挑选，甚至添加自定义模块。 如果要使用FRAME开发项目，Substrate Developer Hub Node Template 模板可作为一个很好的起点。 要了解更多信息，请参见教程构建一个 dApp 和添加一个Pallet.。

  3. 使用 Substrate Core：可跳过整个 FRAME，并且可以从头开始设计和实现整个 Runtime。 任何能编译成 WebAssembly的语言 ，均能用来实现这个方案。 只要 runtime 兼容 Substrate 节点的抽象区块创建逻辑，那么你就可以简单地从 Wasm 二进制代码块中构造一个新的创世区块，然后用现有的、基于 Rust 的 Substrate 客户端启动你的区块链。 若不兼容，那么你需要改变客户端的区块创建逻辑，甚至可能需要改变区块头和区块序列化格式。 就开发强度而言，这是迄今为止使用 Substrate 最困难的方式，但也给了你最大的自由度。
