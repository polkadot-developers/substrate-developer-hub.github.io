SCALE 编解码器

SCALE（Simple Concatenated Aggregate Little-Endian）编解码器是一个轻量级、高效的二进制序列化和反序列化编解码器。

它是专为资源受限的执行环境所设计的，比如 Substrate runtime，可对数据进行高性能、无需拷贝的编码和解码。 它无法进行自我描述，并假定解码环境拥有编码数据的所有类型信息。

Substrate 中的 SCALE

Substrate 使用 parity-scale-codec，它是 SCALE 编解码器的 Rust 实现。 采用这个库和 SCALE 编解码器会使 Substrate 和区块链系统会更有优势，因为：

  • 相对于 serde 这样的通用序列化框架来说，它是轻量级的，因为 serde 添加了大量的模板，会使二进制文件过大。
  • 它不使用 Rust STD，因此可以把文件编译为 Wasm格式，用在 Substrate runtime 之中。
  • 它对在Rust中为新数据类型派生编解码逻辑的支持极为友好： 
    #[derive(Encode, Decode)]

相对于重用现有的 Rust 编解码器库而言，定义出 Substrate 上使用的编码方案非常重要，因为这个编解码器需要在其他支持互操作性的平台和语言上重新实现。

编解码器定义

在此，你会了解 SCALE 编解码器是如何对不同类型进行编码的。

定宽整数

基础整数类型是由定宽的little-endian (LE) 编码格式进行编码的。

示例

  • 有符号的8位整数 69: 0x45
  • 无符号的16位整数 42: 0x2a00
  • 无符号的32位整数 16777215: 0xffffff00

紧凑/通用整数

紧凑型或通用整数编码对于编码大整数 (最大2**536) 来说已经足够了，而且对大多数值的编码会比定宽版本更有效率。 (虽然对于单字节值来说，定宽整数编码会更有效些。)

它用两个最小有效位进行编码以表示模式：

  • 0b00：单字节模式；前面6位是数值的LE编码 (只对0-63的数值有效)。
  • 0b01：双字节模式：前面6位和下一字节是值的LE编码(只对值 64-(2**14-1) 有效)。
  • 0b10：四字节模式：前面6位和接下来的3个字节是该值的LE编码(仅对值 (2**14-1)-(2**30-1) 有效)。
  • 0b11：大整数模式。前面6位表示的值是后面的字节数减4。 值以 LE 编码形式包含在后面的字节中。 最后 (最重要的) 的字节必须为不能为零。 仅对值 (2**30-1)-(2**536-1) 有效。

示例

  • 无符号整数 0: 0x00
  • 无符号整数 1: 0x04
  • 无符号整数 42: 0xa8
  • 无符号整数 69: 0x1501

错误：

  • 0x0100: 模式1中对0这个值的编码

布尔型

布尔型值使用单个字节的最小有效位进行编码。

示例

  • 布尔值 false: 0x00
  • 布尔值 true: 0x01

Option类型

特定类型的一个或零个值。 编码为：

  • 如果是 None ("empty" 或 "null")，则为0x00 。
  • 0x01 后面加上编码值 Some。

一个例外是，如果是布尔型，那么它的大小总是一个字节。

  • 如果是 None ("empty" 或 "null")，则为0x00 。
  • 如果是 true，则为0x01 。
  • 如果是 false 值，则为0x02 。

Results类型

Results是常用的枚举类型，用于表示某些操作是成功还是不成功。 编码为：

  • 如果操作成功，0x00后面再加上编码值。
  • 如果操作不成功，0x01 后面再加上错误编码。

示例

// A custom result type used in a crate.
let Result = std::result::Result<u8, bool>;
  • Ok(42): 0x002a
  • Err(false): 0x0100

数组(列表，序列，集合)

对同类型值的集合进行编码时，在编码前加一个 compact 前缀，然后依次将每个项的编码连起来。

示例

16位无符号整数的数组：

[4, 8, 15, 16, 23, 42]

SCALE 编码后的字节：

0x18040008000f00100017002a00

字符串

字符串是包含有效的 UTF8 序列的数组。

元组

一个固定大小的数值序列，每个数值都可以是不同类型，但这些类型是预先确定且不可更改的。 编码时就是简单把所有值的编码连接起来。

示例

无符号整数和布尔值组成的元祖：

(3, false): 0x0c00

数据结构体

对于结构体而言，虽然值被命名了，但编码并不使用该名字 (名字是被忽略的 - 仅值的顺序重要)。 所有容器都是连续存储元素的。 元素的存储顺序取决于容器，而不是一成不变的，因此不能依赖于解码。

这就意味着，如果将某个字节数组解码成一个指定的结构体（结构体本身是排好序的），然后再重新编码，可能会导致新编码的数组与原有的字节数组不一样。

示例

想象一个 SortedVecAsc<u8> 结构体，它的元素总是按升序排列，你有 [3, 5, 2, 8]序列，其中第一个元素是后面跟随的元素的数量(即 [3, 5, 2] 将是无效的结构)。

SortedVecAsc::from([3, 5, 2, 8]) 会将其解码为 [3, 2, 5, 8]，这与原始编码不符。

枚举(打标的并集)

固定数量的变量，每一个变量都是互斥的，并可能还会包含另一个或另一系列的值。

编码后的第一个字节表示值所在的变量的索引。 后续的字节是该变量所包含的实际值的编码结果。 因此，最多支持256个变量。

示例

enum IntOrBool {
  Int(u8),
  Bool(bool),
}
  • Int(42): 0x002a
  • Bool(true): 0x0101

实现版本

Parity SCALE 编解码器已经用许多语言实现了，包括用 Rust 编写的参考实现，这是由 Parity Technologies 维护的。

  • Rust: paritytech/parity-scale-codec
  • Python: polkascan/py-scale-codec
  • Golang: itering/scale.go
  • C++: soramitsu/scale
  • JavaScript: polkadot-js/api
  • AssemblyScript: LimeChain/as-scale-codec
  • Haskell: airalab/hs-web3
  • Java: emeraldpay/polkaj
  • Ruby: itering/scale.rb
