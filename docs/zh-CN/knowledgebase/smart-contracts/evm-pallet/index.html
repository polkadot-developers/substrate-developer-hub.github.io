EVM 模块

FRAME EVM 模块 提供了一个 EVM 执行环境，允许在基于 Substrate 的区块链中执行无需修改的 EVM 合约代码。

EVM 引擎

The EVM module uses SputnikVM as the underlying EVM engine. 此引擎作过大的重写，现在更加 模块化。 日后我们希望允许用户更换类似 gasometer 这样的组件，并加入自己的自定义组件。

执行周期

EVM模块管理了一套单独的账户集。 基于 Substrate 的账户可以调用 EVM 模块，将资产从 Substrate 基础代币的余额存入或取出到另一个由 EVM 模块管理或使用的账户中。 一旦用户存入了自己的余额，就可以使用这个模块创建和调用智能合约。

Substrate 账户和 EVM 外部账户是一一对应的，该映射关系由转换函数定义。

EVM 模块 vs Ethereum 网络

与 Ethereum 主网相比，EVM 模块应该能够产生几乎相同的结果，包括 gas 成本和余额变化。

可观察到的差异包括：

  • 取决于 Substrate Runtime 系统模块的设置，可用的区块哈希长度可以不是256位。
  • 在这个模块中 Difficulty 和 coinbase 是没有意义的，目前被硬编码为零。

我们目前并不打算让state root等不可观测的行为变得相同。 我们也不打算遵循完全相同的交易/收据格式。 然而，给定一个 Ethereum 交易和一个 Substrate 账户的私钥，我们应能将任何 Ethereum 交易转换为与该模块兼容的交易。

目前 gas 的配置硬编码在伊斯坦布尔硬分叉中。 以后可以扩展到支持早期的硬分叉配置。
