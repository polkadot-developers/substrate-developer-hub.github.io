合约 (Contracts) 模块

The Contracts pallet provides the ability for the runtime to deploy and execute WebAssembly (Wasm) smart contracts.

Wasm 引擎

合约模块依赖于 Wasm 的沙盒接口，它定义了 Runtime 内可用的 Wasm 执行引擎。 现在是实现于 Wasm 解释器 wasmi 里。

功能

合约模块在智能合约的部署和执行上有许多合约开发者熟悉的功能以及一些新功能。

基于帐户

合约模块采用的是类似于现有很多智能合约平台的账户体系。 对 Substrate Runtime 来说，合约账户就像普通的用户账户一样；但是，除了普通账户所拥有的 AccountID 和 Balance 之外，合约账户还有相关的合约代码和一些持久的合约存储。

两步部署

用 Contracts 模块部署合约需要两个步骤：

  1. 在区块链上存储 Wasm 合约。
  2. 开启一个拥有存储，与该智能合约挂勾的新帐户。

这意味着可以使用同一个 Wasm 代码初始化多个具有不同的构造参数的合约实例，从而减少区块链上 Contracts 模块所需的存储空间。

合约调用

调用合约可以改变合约内的储存、创建新合约，或因而调用其他合约。 因为 Substrate 为您提供了编写自定义 Runtime 模块的能力，Contracts 模块也让您可用合约账户直接异步调用那些 Runtime 函数。

沙盒保护

合同模块旨在供公共网络上的所有用户使用。 这意味着合约只能直接修改他们自己所拥有的存储。 为了给底层区块链状态提供安全保障，合约模块实现了可逆交易，可回滚那些对存储进行改动而没有成功完成的合约调用。

手续费

为了限制一次交易可使用的计算资源，合同调用是要收取手续费 (gas fee)。 在构造合约交易时，得须指定 gas 限额。 随着合约的执行，gas 根据计算的复杂性逐步被消耗。 如果在合约执行完成前达到 gas 限额，则交易失败，合约存储被还原，gas 费用 并不会 退还给用户。 如果合约执行完成时还有剩余 gas，则在交易结束时退还给用户。

Contracts 模块决定 gas 的价格，即 Substrate weight system 和单个 gas 之间的换算。 因此，要执行一项交易，用户须要有至少 gas price * gas limit 的可用余额。

存储租用

于 gas 限制了对交易所使用的计算资源类似，存储租金限制了合约在区块链存储中的占用率。 合约账户按其使用的存储量所占的比例支付租金。 当合约可用余额低于某个限额时，合约账户会变成一个 “墓碑”，其存储被清空。 墓碑合约可通过追加使它激活的最少资金和提供变成墓碑合约时所清掉的数据，来重新激活合约。

Contracts 模块与 EVM 的关系

Contract 模块是在当前智能合约生态系统中的现有理念进行了迭代，尤其是 Ethereum 和 EVM。

Contracts 模块和 EVM 之间最明显的区别是用于运行智能合约的底层执行引擎。 EVM 理论上是很好的执行环境，但要使用在现代硬件上却并不实用。 例如，在现代体系架构上操作 256 位元的整数要比操作标准类型复杂得多。 以太坊团队自己也在为下一代网络调研使用 Wasm 的可能性。

EVM 只在储存发生时收取储存费。 这种一次性的成本导致区块链上的一些存储空间被 永久 占用，这在经济上很不合理的。 本合约模块尝试通过 storage rent 来修复这一点，以确保在区块链上持续存在的任何数据都会因对资源的占用被适当收费。

合约模块选择使用 两步过程 来创建合约，这从根本上改变了合约在链上的存储方式。 合约地址、占用储存、和余额现在与基础合约逻辑进行了分离。 这允许了类似于以太坊的 create2 行为，甚至允许于 Substrate 区块链上的合约，可进行修复和级合。
